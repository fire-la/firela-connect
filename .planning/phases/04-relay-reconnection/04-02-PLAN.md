# Current Implementation Analysis
Current `packages/core/src/relay/client.ts`:
- `pendingEvents` Map stores unacknowledged events but is cleared on disconnect
- No mechanism to request missed events after reconnection
- No `lastEventTimestamp` tracking for recovery

# State Recovery Requirements
1. Track last successfully processed event timestamp
2. On successful reconnection, send recovery request to server
3. Server responds with events missed during disconnect
4. Process recovered events through normal handler flow

**Protocol Extension Needed:**
- Client → Server: `event_recovery` message with `lastEventTimestamp`
- Server → Client: `event_recovery_response` with missed events

**Key source files:**
@packages/core/src/relay/client.ts
@packages/core/src/relay/types.ts

**Depends on:** 04-01-PLAN.md (Full Jitter implementation)

**Tech stack available:**

- RuntimeContext pattern for abstraction
